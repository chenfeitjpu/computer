# 使用 #
## 基础 ##
```
分布式会出现的问题
- 网络一定会出问题
  延迟、丢包、乱序、重复、分区
- 节点一定会挂
  进程crash、机器宕机、GC停顿、STW
- 时间是不可信的
  没有全局时钟，时间只能"近似"
```

## 三大理论 ##
```
CAP定理
- C 一致性(读到最新值)
  所有用户在同一时间看到的数据都是一致的，你在节点A写入一条数据后，马上在节点B读到的结果一定包含这次写入
- A 可用性(每次请求都有响应)
  每一个收到的请求都能在有限时间内得到响应，即使这个响应可能不是最新的数据，不管有多少节点宕机，只要请求打到的节点还活着，它就必须给你「某个有效结果」
- P 分区容错(网络分区仍能工作)
  系统可以在「网络分区」的情况下继续工作。集群被网络故障切成若干块，各块之间通信中断或非常不可靠，但每块内部通信还正常。

BASE理论
- Basically Available(基本可用)
  系统在出故障或高压下，允许功能/性能降级，但整体仍然可用
- Soft State(软状态)
  系统允许存在一段时间的中间状态 / 不一致状态，不要求随时都是强一致
- Eventually Consistent(最终一致性)
  在没有新的更新操作后，经过一段时间，所有副本最终会收敛到一致的状态

一致性模型
- 强一致
- 顺序一致
- 因果一致
- 最终一致
```
