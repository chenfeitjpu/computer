## 设计原则
```
solid
- 单一职责
  定义：一个类应该只有一个引起它变化的原因
  理念：每个类只负责一件事情
- 开闭原则
  定义：软件实体应该对扩展开放，对修改关闭
  理念：通过扩展而非修改来实现新功能
- 里氏替换
  定义：子类对象必须能够替换掉所有父类对象
  理念：继承必须确保父类所拥有的性质在子类中仍然成立
- 接口隔离
  定义：客户端不应该依赖它不需要的接口
  理念：一个类对另一个类的依赖应该建立在最小的接口上
- 依赖倒置
  定义：高层模块不应该依赖低层模块，两者都应该依赖抽象；抽象不应该依赖细节，细节应该依赖抽象
  理念：面向接口编程，而非面向实现编程

DRY(Don't Repeat Yourself)
系统中的每一处知识都必须有单一、明确、权威的表示

KISS(Keep It Simple, Stupid)
保持简单和愚蠢

YAGNI(You Aren't Gonna Need It)
不要实现你目前不需要的功能
```
