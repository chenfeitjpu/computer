## 简介
```
变量
- 本质是名字(name)到对象(object)的绑定(binding)，变量不存储对象
- is比较的是对象身份(内存地址)是否相等，==比较的是对象内容(eq)是否相等

命名空间&作用域
- 命名空间  名字到对象的映射表  定义时确定
  - 局部命名空间  locals()
  - 全局命名空间  globals()
  - 内建命名空间  builtins
- 作用域  变量在程序中可以被访问的范围
  - L  Local
  - E  Enclosing
  - G  Global
  - B  Builtins
```

## 数据类型
```
分类
- 基本数据类型
  - int
    - 小整数缓存(-5~256)
  - float
  - bool
  - str
    - 格式化  %、format、f
    - 字符串驻留
      - 标识符(变量名/属性名/函数名)
      - 子面量字符串
  - None
    - is None比较的好处
      - 语义明确
      - 安全，不会被__eq__重写
      - 性能更高，没有__eq__函数调用
- 复合数据类型
  - list
    - 列表推导式
      优雅、简洁的创建列表  [表达式 for 变量 in 可迭代对象]
    - range
      惰性生成数字序列  range(start, stop, step)
  - tuple
    - 更省内存
      - 简化结构，去掉了可变属性(当前大小，分配容量、数组指针)
      - 不需要预留空间
      - 内存连续
      - 可hash
  - set
  - frozenset
  - dict

可变 & 不可变
- 不可变类型
  对象创建后，其内容不能改变，对不可变对象的操作会创建新对象
- 可变类型
  对象创建后，其内容可以改变，对可变对象的操作修改原对象
  list、set、dict

函数
- PyFrameObject(栈帧)
  - 一次Python函数调用的执行环境
  - 函数的执行状态在函数返回后仍然存在(放在堆上)
- 参数
  - 默认参数在函数定义时就会被计算并保存到函数对象上(__defaults__)
  - 传参是引用传递
- 匿名函数  可以接受任意多个参数并且返回单个表达式值的函数
  - lambda 参数: 表达式
- 高阶函数  入参或者返回值是参数
- 偏函数  固定函数的某些参数，生成一个新的函数
  - 暂存参数，和调用参数合并，传给原函数
- 闭包  引用了自由变量(函数内部没有定义的变量)，自由变量存储在cell对象

迭代器
迭代器是一个可以记住遍历状态的对象
- 实现
  - __iter__()  返回迭代器对象本身
  - __next__()  返回下一个元素
- 和可迭代对象区别
  - 可迭代对象  不维护迭代状态
  - 迭代器  记住遍历状态

生成器
一种特殊的迭代器，可以按需生成值，而不是一次性创建所有值
- 实现
  - yeild
  - 生成器表达式 (表达式 for 变量 in 可迭代对象)

装饰器
在不修改原有函数代码的情况下增加函数的功能，使用@修饰
- 分类
  - 无参数装饰器  两层函数声明
  - 带参数装饰器  三层函数声明
- 常见装饰器
  - functools.wraps(func)  保证被装饰的函数元信息(__name__等)不丢失
  - functools.lru_cache  对相同参数的调用直接返回缓存结果，避免重复计算

类
- 类创建过程
  - metaclass(类名，基类元组，命名空间)
- 实例创建过程
  - 调用__new__()，创建实例
  - 调用__init__()，初始化实例
- 元类(metaclass)
  用来创建类的类，元类可以控制类的创建过程
- 常见属性&方法
  - __slots__  用于限制实例属性并避免创建实例__dict__，降低内存占用
  - getattribute  所有属性访问都会触发
  - getattr  仅当属性找不到时才触发
- 继承
  - MRO(方法解析顺序)
    - C3线性化算法 
      - 性质
        - 局部优先级  子类声明的父类顺序必须被保留
        - 单调性  子类不能改变父类MRO的顺序
      - merge算法
        - L[C] = [C] + merge(L[A], L[B], ..., [A,B])
        - 在merge过程中，每次选择一个不出现在其他序列tail中的头元素加入结果
  - super  
    - super(currentClass, instance)
    - 调用的是MRO中的下一个类，实现协作式多继承
    - 直接调用父类，多继承容易重复调用
- 多态
  - 鸭子类型多态  不关心类型，只关心行为，多个类都实现相同的方法
  - 协议多态  协议定义了一组方法/属性签名，任何实现了这些签名的类都自动符合该协议
  - OOP多态  子类继承父类并重写方法
```

## 数据结构
```
PyObject
- 结构
  typedef struct {
    Py_ssize_t ob_refcnt;
    PyTypeObject *ob_type;
  } PyObject;
- 引用计数
  - 赋值只是增加引用
  - del只是减少引用
- 对象共享导致数据污染
- 减少对象创建减低GC压力

dict
- 哈希冲突解决
  开放寻址法
  - 减少对象指针，降低内存占用
  - 提高cpu cache命中率
  - 更高的查找性能，负载因子2/3，保证有较多的空位，查找使用了跳跃算法
- 如何保证插入顺序
  - indices(索引数组)，无序
  - entries(条目数组)，有序
- 元素删除
  为了保证有序性，标记删除，当被删除元素过多时，dict会重建哈希表
```

## 内存管理
```
浅拷贝&深拷贝
- 浅拷贝  只复制对象本身，不会递归复制对象内部的嵌套对象  copy.copy()
- 深拷贝  不仅复制对象本身，也递归复制对象内的所有嵌套对象，创建所有对象的完整副本  copy.deepcopy()
对于不可变对象拷贝产生的都是原对象的引用

垃圾回收(同步)
- 引用计数(主机制)
  - 每个对象维护引用数量，引用为0时立即释放，无法处理循环引用
  - 增加  变量赋值、容器内添加元素、函数传参、返回值绑定
  - 减少  变量重新赋值、容器内删除元素、del、变量离开作用域
- 分代垃圾回收(辅助机制)
  - 仅处理被GC追踪的容器对象，采用三代分代策略(0/1/2)，新对象进入第0代，经一次回收存活，晋升到第1代，再存活，晋升到第2代
  - 回收过程
    - 自动触发
      - 0代(新增对象-回收对象大于阀值)
      - 1代(0代回收次数超过阀值)，高版本已去掉
      - 2代(1代回收次数超过阀值)
    - 扫描该代及更年轻代的tracked对象，复制refcount到gc_refs，扣除内部引用，找出gc_refs == 0的不可达候选对象，进行可达性传播修正(从gc_refs > 0的对象出发，把它们能到达的对象从不可达候选集里挪出去)
    - 统一调用finalizer，断开引用，通过引用计数释放对象
```

## 并发控制
```
异步模型
- 执行流程
  - 执行协程
  - 遇到await
  - 暂停协程
  - 注册Future
  - 事件循环调度其他任务
  - Future完成
  - 恢复协程
- async + await
  - async def返回coroutine对象
  - await会暂停当前协程，把执行权交给事件循环，等事件完成后再恢复执行

GIL(全局解释器锁)
GIL是一个互斥锁（mutex），它确保在任何时刻，只有一个线程可以执行Python字节码
- 作用
  - 保证了引用计数的线程安全，解决内存管理的线程安全问题
- 绕过GIL的限制
  - 使用多进程（multiprocessing）
```
