## 简介
定义
RPC(Remote Procedure Call)远程过程调用在一台机器上像调用本地函数一样去调用另一台机器上的函数
RPC本质上是在TCP+HTTP/2+序列化之上，把接口契约(请求、响应的数据结构)、调用语义(错误处理、超时、重试)和服务治理整合成了一套完整的解决方案

## 使用
```
组成
- client
- client stub(代理)
- server
- server stub(代理)

调用过程
- client发起调用
- client stub处理(序列化+寻址+发送)
- server stub处理(接收+反序列化+调用服务)
- server stub序列化结果并发送
- client stub接收并反序列化结果
- client得到最终结果
```

## 常见问题
```
RPC和HTTP对比
- HTTP是通信协议，强调资源与动作，解决数据怎么在两端之间传
- RPC是调用模型，强调用服务与方法，解决怎么优雅地调用远程方法
```
