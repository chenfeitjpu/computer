# 定义 #
```
数据结构是为了高效地支持一组操作，而设计的数据组织方式和存储方式
不同的数据结构，核心区别在于：支持哪些操作更高效，以及它们的时间、空间复杂度
```

# 使用 #
## 物理存储方式 ##  
```
- 顺序存储：比如数组，使用连续内存，按下标随机访问是O(1)，但在中间插入和删除一般是O(n)
- 随机存储：比如链表、用指针把节点串起来，已知位置时插入删除是O(1)，但查找节点通常要O(n)
```

## 逻辑组织方式 ##  
```
- 线性结构：数组、链表、栈、队列，特点是元素前后有线性顺序
- 非线性结构：树，图
- 其它高级数据结构：字符串、哈希表、布隆过滤器、并查集、LRU、LFU
```

## 操作方式 ##  
```
- 增、删、改、查、遍历
```

## 设计思路  
```
思路
- 抽象  先把问题抽象成"需要支持什么操作、数据之间什么关系"
- 选择/转换  根据操作和复杂度要求，选择合适的数据结构，必要时把数据转换成更适合的形式
- 剪枝  利用结构特性做预处理、剪枝
- 复用  使用缓存，尽量复用已有的数据
```
